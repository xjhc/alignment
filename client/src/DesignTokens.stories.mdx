# Design Tokens

Our design system is built on a foundation of design tokens that ensure consistency across all components and interfaces. These tokens are automatically generated from JSON definitions using Style Dictionary.

## Usage

### CSS Variables

All tokens are available as CSS custom properties:

```css
.my-component {
  background-color: var(--bg-primary);
  color: var(--text-primary);
  padding: var(--space-4);
  border-radius: var(--radius-md);
  font-family: var(--font-sans);
}
```

### TypeScript/JavaScript

Tokens can also be imported and used in JavaScript:

```typescript
import { tokens } from './styles/generated/tokens';

const styles = {
  backgroundColor: tokens.theme.dark.bg.primary,
  color: tokens.theme.dark.text.primary,
  padding: tokens.spacing[4],
  borderRadius: tokens.border.radius.md,
};
```

## Maintenance

This documentation is automatically synchronized with the token definitions. All tokens are:

1. Defined in JSON files in the `/tokens` directory
2. Compiled by Style Dictionary into CSS variables and TypeScript exports
3. Automatically updated when the design system changes

To add or modify tokens, edit the JSON files in `/tokens` and run `npm run build:tokens`.
